Pygame2exe打包的EXE运行不成功，Runtime Error!

    今天这个问题困扰了我整整一天！奇怪之前一直可以打包成功的，突然出现了这个问题。想想可能是因为我调用了TextWidget（参见TextWidget页面）的缘故，因为我调用之前的那个版本是可以打包成正常运行的exe的。


    在确认了是TextWidget调用的问题后，拼命地到TextWidget里面寻找答案，猜测是不是某个变量（比如event）的赋值在不同版本的python之间运行会出现差异，猜测是不是因为缺少了什么Win32运行时库，怎么别人也有调用TextWidget的代码却能正常打包，然后猜测是不是我和他的pygame2exe.py不一样。拿selsine发布的TextWidget与网上找到的调用的TextWidget比较了一下，虽然刚开始就在众多些微的差别中发现一处不同的地方，但是我对之付诸一笑毫没在意：

self.size = size
self.font_filename = font_filename

这两句出现的顺序不一样。先赋值size和先赋值font_filename有什么区别嘛。想当然的也就没去管它。


    直到我用二分法逐步寻找出错的地方时才发现，原来这个size的赋值操作是有问题的！

    不过，这样说是有针对性的，就是说，这个size的赋值操作本身是没有错的，但是如果用pygame2exe来打包，那么这里就存在一个隐藏的很难发现的问题，会导致最后打包出来的EXE文件运行时候出现Runtime Error : This application has requested the Runtime to terminate it in an unusual way. Please contact the application's support team for more information.

    这个问题就是Pygame2exe页面里面提到的font问题。为了用pygame2exe得到正常运行的exe，font不能用None，必须指定一个font文件，并把它放到exe能找到的位置。Pygame2exe页面里面提到了freesansbold.ttf的例子。

  然后我们在回到当前遇到的size赋值操作。这里TextWidget是一个new-style classes,size是一个property，指定了size会触发__set_size(self, size)，然后这里面又会调用create_font。前面提到我认为“先赋值size和先赋值font_filename是没什么区别的”，但实际上问题就出在这里。如果先赋值size，那么create_font的时候会因为没有filename而用None生成font，虽然后面赋值font_filename后又显示的调用了create_font，但是用None来生成font的这一步操作并没有跳过，所以导致了EXE程序的Runtime Error；而先赋值font_filename就不会存在这样的问题！所以，为了让pygame2exe生成正确的exe，还是必须要先设置font_filename！

  于是乎，这成了TextWidget的一个bug，或者至少是一个可以改进的地方（如果不考虑用pygame2exe打包的话）。
